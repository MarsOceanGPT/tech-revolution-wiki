# GitHub Copilot：AI编码助手

> 2021年6月，GitHub和OpenAI推出了Copilot，一个AI编码助手，建议代码片段和完整函数。开发人员描述他们想要什么，Copilot自动生成代码。初始测试表明Copilot将生产率提高了55%。到2023年，Copilot有超过100万个付费订阅者。它改变了开发人员如何编码。

## 故事

Copilot由OpenAI的Codex（一个在互联网代码上训练的大型语言模型）提供支持。GitHub用于支持代码，提供了超过数十亿行代码进行训练。OpenAI使用这个数据创建Codex。Codex可以根据自然语言描述或代码上下文生成代码。

GitHub将Copilot与Visual Studio Code集成。当开发人员开始输入时，Copilot建议下一行代码。开发人员可以接受、拒绝或修改建议。这类似于Google搜索的自动完成，但适用于代码。

Copilot的影响是深刻的。开发人员可以更快地编码。重复的代码可以生成而不是手动输入。开发人员可以尝试新的库或语言，而Copilot提供了建议。生产率提高。

但也有担忧。Copilot是在包括GitHub中的所有代码上训练的。有些代码有开源许可证（MIT、GPL等）。Copilot有时会复制训练数据中的代码。这引发了版权问题。开发人员起诉GitHub，声称Copilot在没有许可的情况下使用他们的代码。案件仍在进行中。

## 分析

### 维度归属

**B-基础设施**：Copilot代表了开发者基础设施的新一代。不再只是IDE（集成开发环境）和库，现在是AI助手。这改变了开发人员的工作方式。他们变成了提示工程师而不是代码编写器。

**E-技能断裂**：一些开发人员认为Copilot威胁了编程技能。如果AI生成所有代码，开发人员将失去学习编程的机会。但其他开发人员认为Copilot解放他们专注于更高级别的设计。技能改变。旧的编码技能变成了低价值。新的AI协作技能变成了高价值。
