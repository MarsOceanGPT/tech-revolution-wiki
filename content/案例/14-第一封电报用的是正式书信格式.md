# 第一封电报用的是正式书信格式

> 第一条电报被发送时，人们用"尊敬的先生"开头。电报按字数收费。所以"尊敬的先生"花了4个字的钱。这太浪费了。35年后，发明了一个3000词的缩写代码。这是现代文本通信的诞生——用简写而不是完整词语。

## 故事

1844年，Samuel Morse发送了第一条电报："What hath God wrought"（上帝做了什么）。

但日常的电报是什么样的？完全像书信：

```
尊敬的先生，

我写这封信是向您汇报...
[正式书信的全部内容]

您忠诚的仆人，
John Smith
```

为什么？因为没有人知道怎么用电报。他们假设电报就是超快速的信件。

电报的成本是按字数收费的。在1850-1870年间，一条长距离电报可能要花很多钱。所以人们开始自然地缩写：

```
"尊敬的先生" → "亲爱的先生" → "先生" → 最后干脆不写
"我写这封信是向您汇报" → "汇报如下"
```

但这还不够。

**1879年，Walter Phillips（当时在Associated Press工作）编写了《Phillips Code》——一个有3000多个代码的缩写系统。**

每个常见的短语被分配一个2-3位的数字代码。例如：
- "Congress" = "AE"
- "Amendment" = "AF"
- "Adjournment" = "AG"
- "Appropriation" = "AH"

一份原本需要200个字的新闻稿，可以被压缩成50个字的代码。

效果很明显。Phillips Code成为了标准。报纸、通讯社、商人——所有人都用它。

最有趣的是：一些Phillips Code的缩写直到今天仍在使用。例如：
- **POTUS** (President Of The United States) — 1895年加入
- **SCOTUS** (Supreme Court Of The United States) — 1879年就有了

这是什么？这是**现代短语文化的原型** — LOL、ASAP、FYI等等都来自这个传统。电报的媒介约束（按字收费）创造了一种新的文本简写方式。

时间线：
- 1844-1860：电报 = 格式化的书信
- 1860-1879：电报操作员开始创造自己的缩写
- 1879：Phillips Code官方发布，3000个代码
- 1879-1920：Phillips Code被广泛采用，甚至影响了电话和无线电通信
- 1920-1950：随着电话和无线电的普及，Phillips Code逐渐衰退
- 1960-现在：互联网文本通信（短信、邮件、即时消息）重新发明了这种简写文化

## 分析

### 维度归属

这不是"范式滞后"，而是**"媒介约束激发创新"**。电报的按字计费特性，激发了操作员和编码者创造一种完全新的文本表达方式。

这种约束导致了积极的创新，而不是消极的拖延。

### 跨时代类比

1. **短信和140字限制** — Twitter的140字限制（后来280字）创造了一种新的简洁文体。现在这种简洁性成为了社交媒体的标志。

2. **电子邮件主题行** — 限制长度的主题行创造了一种新的"标题文体"。

3. **现代编程** — 代码变量的简写（如i、j、tmp）来自早期计算机的内存限制。现在即使内存充足，程序员仍然倾向于使用简短的变量名。

4. **Markdown语法** — 为了适应纯文本格式，Markdown发明了一种新的排版语法（#标题、**粗体**）。这种语法现在被广泛采用。

### 为什么重要

这说明：**技术约束往往不是限制，而是创意的引擎**。最伟大的一些创新（简写代码、短消息格式、表情符号）来自于对媒介约束的适应。
