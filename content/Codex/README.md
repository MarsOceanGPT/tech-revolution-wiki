# Codex 评审包

这次评审不改动原有知识库内容，只把意见和建议写在这里。

建议阅读顺序：

1. [[项目评估与建议]]：先看整体判断、优势、风险、优先级。
2. [[模式层与 Wiki 化建议]]：再看如何从“素材库”走向“可浏览的百科”。
3. [[趣味案例策展建议]]：最后看如何把你最想要的“有趣、小而巧、可互证”的案例做成亮点。

本次评审基于以下观察：

- 项目现有规模已经不小：`案例 125`、`死胡同 10`、`时代 8`、`维度 7`
- 当前 `draft` 明显多于 `published`：扫描结果约为 `127 draft / 1 published`
- `模式/` 目录目前为空，说明“模式层”仍未真正落地
- 多个时代页仍有 `*待补充*` 占位
- 抽样检查发现少量链接目标尚不存在，例如：
  - `[[模式：三代范式迁移]]`
  - `[[模式：基础设施遗产]]`
  - `[[互联网早期虚拟社区文化]]`
  - `[[《The AOL Effect》]]`
  - `[[《The Steam Engine》]]`

这几个事实不说明项目有问题，反而说明你已经完成了最难的第一步：把一个强主题做成了真正有密度的素材库。现在最值得投入的，不是继续单纯铺量，而是把这些内容变成一套读者能够“沿着模式不断跳转”的体验。
